システム要件定義書プロジェクト名: Old iPad Smart Displayization Projectバージョン: 1.0作成日: 2025年12月9日1. プロジェクト概要1.1 目的使用頻度の低下した旧世代iPadを、常時表示可能な「スマートディスプレイ（置き時計兼情報端末）」として再利用する。1.2 コンセプトインテリアに馴染む「デジタルフリップ時計」をメインとし、生活に必要な最低限の情報（天気・日付）をスタイリッシュに表示する。2. システム構成・環境2.1 ターゲットデバイスハードウェア: iPad（旧モデル想定）OS: iOS（Safariブラウザ利用）設置状況: 横向き（Landscape）、常時給電2.2 開発環境・技術スタック言語: HTML5, CSS3, JavaScript (Vanilla JS - フレームワークなし)ホスティング: GitHub Pages (推奨) または ローカルファイル実行外部API: OpenWeatherMap API (Free Tier)3. 機能要件 (Functional Requirements)システムが実装すべき具体的な機能。ID機能カテゴリ機能名詳細仕様F-01時計現在時刻表示「時（HH）」と「分（MM）」を2桁表示する。F-02時計秒更新1秒単位で内部時刻を更新する（表示は分単位で可）。F-03時計フリップアニメーション分が切り替わる際、パタパタ時計風の視覚効果（または擬似デザイン）を持たせる。F-04カレンダー日付表示「YYYY年MM月DD日（曜日）」の形式で表示。日本語ロケール対応。F-05天気天気情報取得OpenWeatherMap APIを使用し、指定地域の天気を取得する。F-06天気表示項目天気アイコン、気温（摂氏）、天気詳細（「晴れ」「曇り」など）を表示。F-07天気自動更新API制限を考慮し、10分〜30分間隔でデータを再取得する。4. UI/UX要件 (User Interface / User Experience)画面のデザインや操作性に関する要件。UI-01 テーマ: ダークモード（背景色 #000）。夜間の眩しさを軽減し、没入感を高める。UI-02 レイアウト: 画面中央に大きく時計を配置し、視認性を最優先する。天気・日付は補助情報として配置。UI-03 フォント: デジタルガジェット感のある等幅フォント（Roboto Mono等）を採用。UI-04 操作制限:スクロール禁止（overflow: hidden）ズーム禁止（user-scalable=no）アドレスバー非表示（ホーム画面追加時のPWA挙動）5. 非機能要件 (Non-Functional Requirements)性能や保守性に関する要件。NF-01 パフォーマンス: 旧世代iPadでもカクつかずに動作すること（軽量なDOM操作）。NF-02 可用性: ネットワーク切断時でも時計機能は止まらず動作し続けること。NF-03 API制限管理: 無料プランの上限（例: 60回/分、1000回/日など）を超えないリクエスト頻度に設定する。NF-04 焼き付き対策: （将来的な検討事項）OLEDモデル等の場合、表示位置を定期的に数ピクセルずらすなどの対策を考慮する。6. データフロー起動: iPadでページをロード。初期化: JSが起動し、端末のシステム時刻を取得して表示。API通信: JSがOpenWeatherMapへリクエスト送信（API Key使用）。描画: 返ってきたJSONデータをパースし、DOM（HTML要素）を書き換え。ループ: setIntervalにより、時計は1秒毎、天気は指定間隔毎に処理を繰り返す。
